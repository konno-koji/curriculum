■ ネットワーク層
IP : internet protocol 相手先のアドレス情報をもとにパケットを送信するプロトコル
IPヘッダ : IPパケットの先頭にある通信制御用の情報を記載した部分のこと
ICMP : internet control message protocol ネットワーク内のノード間で通信状態の確認をするプロトコル
ICMPメッセージ : 通信状態の確認とエラー送信のメッセージを指す
IPアドレス : ネットワーク上の機器に割り当てられる番号
オクテット : 8ビットであること
ネットワーク部 : IPアドレスのうち属しているネットワークを識別する部分。数字が入る部分
ホスト部 : IPアドレスのうちネットワーク内のコンピュータを識別する部分。0の部分
クラス : IPアドレスの構造を分類するもの
プライベートIPアドレス : インターネットの通信には利用できないIPアドレス
グローバルIPアドレス : インタネットの通信に使用可能なIPアドレス
ネットワークアドレス : IPアドレスのホスト部がすべて0のアドレス
ブロードキャストアドレス : IPアドレスのホスト部のビットが1のアドレス
ループバックアドレス : 127.0.0.1のこと。自分自身を示す仮想的なIPアドレス
サブネット : ネットワークを小さく分けたネットワーク
サブネットマスク : ネットワークの範囲を指定する番号
プレフィックス表記 : IPアドレスとサブネットマスクをセットで表記すること
デフォルトゲートウェイ : ネットワークの出入り口。主にルータのこと
IPv6 : IPアドレスを128ビットのデータとして表現する
サブネットプレフィックス : サブネットマスクの1で表記される部分
インターフェイスID : ノードのインターフェイスを識別する名称
ルータ : ネットワーク間を中継する通信機器
ルーティングテーブル : ルータに記憶される経路情報
ネクストホップ : 経路表に記された次のルーターを指す
ARPテーブル : IPアドレスと対応するMACアドレスを記した記憶
ARP : Address Resolution Protocol　IPアドレスからEthernetのMACアドレスの情報を得られるプロトコル
コアルータ : 基幹ネットワークの中心部で用いられるもの。機能性が高い。
センタールータ : 基幹ネットワーク付近で利用されるルータ
エッジルータ : 小規模なネットワーク間で利用されるルータ
ブロードバンドルータ : WANとLANをつなぐルータ
ルーティング : パケット送信時の適切な経路選択
直接接続 : ルータに直接接続するルート情報の登録方法
スタティックルーティング : リモートネットワークを手動で登録する方法
ダイナミックルーティング : リモートネットワークを自動で登録する方法
ディスタンスベクタ型 : ディスタンスとベクトルで最適なルートを決定する方法
リンクステート型 : 詳細なリストを作成してルートを決定する方法
ハイブリッド型 : cisco独自ディスタンスペクタ型を拡張したルート決定方法
■ トランスポート層
コネクション型通信 : 受信が確立した状態で通信する方法
ACK : 段階を踏んで接続を確立すること
コネクションレス型通信 : 受信の確認はせず通信する方法
TCP : 受信確認を行い通信する方法。慎重
スリーウェイハンドシェイク : 送信確認、受信送信確認、受信確認を交互に送り、接続を確立させる方法
Telnet : 遠隔操作するために使用する仕組み
HTTP : ホームページのファイルを転送するときに使うプロトコル
FTP : faile transfer protocol ファイルを転送するときに使うプロトコル
SMTP : simple mail transfer protocol メールを転送するときに使うプロトコル
TCPヘッダ : TCPパケットの先頭部分にある制御情報を記した部分
UDP : user datagram protocol コネクションレス型のプロトコル
RTP : real-time transfer protocol 音声等の連続するデータをリアルタイムに伝送するためのプロトコル
RTCP : RTPの制御をするプロトコル
DHCP :　自動的にIPアドレスを割り振るプロトコル
TFTP :　転送効率を重視しファイルを転送する際に使うプロトコル
RIP : ルーティングにディスタンスベクタ型を使用し最短で指定する
UDPヘッダ : 負荷の軽いUDPパケットの先頭部分にある
